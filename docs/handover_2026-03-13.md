## Completed
- **Refactored `mod_prot_da.R`**: Modularized UI and server logic into top-level helper functions for better maintainability.
- **Tab ID Standardization**: Standardized the proteomics DA tab ID to `"da"` across `mod_proteomics.R` and `mod_prot_da.R`, resolving initialization failures.
- **Fixed Glimma Interactive Plot**:
    - Switched to the modern `Glimma::glimmaXY` API and unified data into a single source of truth (`display_df`) with consistent underscore naming (e.g., `Protein_Ids`). This resolves the DataTables 'Requested unknown parameter' warning.

    - Fixed sample grouping failures in the expression plot by implementing case-insensitive and whitespace-robust matching between the counts matrix and design metadata.
    - Ensured precise alignment between the plot points and the searchable table to fix "no matching records" behavior.
- **PCA Plot Auto-Trigger**: Updated `mod_prot_norm.R` to ensure pre-normalization PCA plots are generated even when entering the tab in advanced normalization states.
- **Design Matrix Preservation**: Resolved a reactive loop issue in `mod_*_design_builder.R` where UI settings were reverting after saving.

## Unresolved

## Decisions Made
- **Tab ID Consistency**: Standardizing on `"da"` for proteomics (consistent with metabolomics/lipidomics patterns) rather than maintaining the `"de"` alias.
- **Modern Glimma API**: Moving away from internal `Glimma:::*` calls to public `Glimma::glimmaXY` to ensure future stability and compatibility with newer versions of the package.
- **Check.Names = FALSE**: Standard practice for all bioinformatics data frames in this project to prevent R's default column name mangling (e.g., spaces to dots) which breaks sample matching.
